	1.	Rules to ALWAYS follow:
        1.	Always check to see if there is an available component in Ingredients 2.0 before creating anything from scratch. 
        2.	Always use semantic variable token names instead of raw hex values, especially for annotations.
        3.	Always use color tokens from the design system for text. 
        4.	Check to see if there are guidelines for the component in the Ingredients 2.0 file before using the component. 
        5.	Typography styles should only be the ones from the Ingredients 2.0 file. 
        6.	Always design with accessibility in mind. We want AA compliance at least. 
        7.	Use auto-layout at all times and do not create “spacing” frames. 
	2.	Buttons
        1.	Use default button size for most scenarios. If using a button inside a small space (like a table row) then use the small size. 
        2.	Button placement - for modals, the buttons should be on the bottom right. The buttons should follow this order [Secondary button][Primary button]
	3.	Spacing
        1.	For buttons, they should be 16px apart on left and right sides. 
        2.	For inputs, select components, and filters, use 16px on left and right sides.
        3.	Use 24px spacing between vertical sections on a page. 
        4.	Follow the built in spacing for components between label and inputs. 
	4.	Text
        1.	Use ‘Body M’ for most body text. If a heading is needed for a paragraph of text, use ‘heading XS’. 
        2.	Do not use ‘Body S’ or ‘Body XS’ unless it is a caption, subtitle, or legal print.
	5.	Headers
        1.	Use ‘heading L’ for the top page header. 
        2.	Use ‘heading S’ for headings within a page
        3.	Use ‘heading XS’ if it is a heading within a paragraph of text.
