## Rules to ALWAYS Follow
1. Always check Ingredients 2.0 before creating anything from scratch.
2. Always use semantic variable token names instead of raw hex values.
3. Always use color tokens from the design system for text.
4. Check guidelines for the component in Ingredients 2.0 before using it.
5. Typography styles should only be ones from the Ingredients 2.0 file.
6. Always design with accessibility in mind. AA compliance at minimum.
7. Use auto-layout at all times. Do not create "spacing" frames.

## Buttons
- Use default button size for most scenarios. Small size only in constrained spaces like table rows.
- Modal button order (bottom right): [Secondary button] [Primary button]

## Spacing
- 16px on left and right sides of buttons, inputs, selects, and filters.
- 24px vertical spacing between sections on a page.
- Follow built-in component spacing between label and inputs.

## Text
- Use 'Body M' for most body text.
- Use 'Heading XS' if a heading is needed within a paragraph.
- Do not use 'Body S' or 'Body XS' unless it is a caption, subtitle, or legal print.

## Headers
- 'Heading L' for the top page header.
- 'Heading S' for headings within a page.
- 'Heading XS' for headings within a paragraph of text.
